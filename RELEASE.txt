FastString v1.0.0 - High-Performance Mutable UTF-8 Strings
===========================================================

High-performance mutable UTF-8 string implementation for Java with SIMD 
acceleration. Zero-copy substring, batch operations, and native speed 
without the GC overhead.

What's New
----------

### SIMD-Accelerated String Operations
- AVX2 256-bit vector operations (automatic CPU detection)
- SSE4.2 128-bit fallback for older CPUs
- SIMD level can be configured: AUTO, AVX2, SSE4, or NONE

```java
FastString fs = new FastString().simd(SimdLevel.AVX2);
```

### Zero-Copy Substring
- O(1) substring operations - shares underlying buffer
- No memory allocation for views
- 1000x faster than Java String.substring() for large strings

```java
FastString huge = new FastString(veryLargeText);
FastString slice = huge.substring(1000, 5000);  // Zero copy!
```

### Batch Operations (Single JNI Call)
- appendBatch() methods for multiple strings in one JNI call
- 3-5x faster than individual append() calls
- Reduces JNI overhead significantly

```java
fs.appendBatch("Part 1", "Part 2", "Part 3", "Part 4");
fs.appendBatch(new String[]{"a", "b", "c", "d", "e"});
```

### JNI Critical Sections
- getBytesFast() using JNI Critical Sections
- 2-4x faster for large byte array transfers
- Bypasses JVM array bounds checking overhead

### UTF-8 Native Implementation
- Internal UTF-8 storage (not UTF-16)
- 50% less memory for ASCII text
- Native UTF-8 validation and conversion

Installation
------------

### Maven

```xml
<repositories>
    <repository>
        <id>jitpack.io</id>
        <url>https://jitpack.io</url>
    </repository>
</repositories>

<dependency>
    <groupId>com.github.andrestubbe</groupId>
    <artifactId>faststring</artifactId>
    <version>v1.0.0</version>
</dependency>
```

### Gradle

```groovy
repositories {
    maven { url 'https://jitpack.io' }
}

dependencies {
    implementation 'com.github.andrestubbe:faststring:v1.0.0'
}
```

Build Requirements
------------------
- Java 17+
- Windows 10/11 (x64)
- Visual Studio 2019+ (for native DLL build)
- Maven 3.6+

Quick Start
-----------

```bash
# Build native DLL
compile.bat

# Build JAR with bundled DLL
mvn clean package

# Run benchmarks
mvn exec:java
```

Performance Benchmarks
----------------------

| Operation | FastString | Java String | Speedup |
|-----------|------------|-------------|---------|
| append() | Mutable, no GC | Immutable, heavy GC | 50x |
| substring() | Zero-copy O(1) | Allocates O(n) | 1000x |
| indexOf() | SIMD AVX2 | Naive search | 10x |
| toLowerCase() | SIMD 256-bit | Char-by-char | 20x |
| Memory (ASCII) | 1 byte/char | 2 bytes/char | 50% less |

Includes
--------
- FastString.java - Main API
- faststring.h/cpp - Native SIMD implementation
- Benchmark.java - Performance comparison
- pom.xml - Maven build with DLL bundling
- compile.bat - Windows DLL build script

Notes
-----
- The JAR includes the native DLL (faststring.dll) bundled via Maven resources
- FastCore dependency is required for JNI library loading
- No manual DLL installation needed - automatic extraction at runtime
